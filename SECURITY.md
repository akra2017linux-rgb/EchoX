# Security Policy

## Reporting a Vulnerability

Se você descobrir uma vulnerabilidade de segurança no EchoX, por favor **NÃO** abra uma issue pública.

Invés disso, por favor envie um email para: **akra2017.linux@gmail.com**

Por favor inclua:
- Descrição detalhada da vulnerabilidade
- Passos para reproduzir (se possível)
- Impacto potencial
- Qualquer sugestão de correção (opcional)

## Política de Segurança

### Versões Suportadas

| Versão | Suportada |
| --- | --- |
| 1.0.x | ✅ Sim |
| < 1.0 | ❌ Não |

### Divulgação Responsável

- Reportes de segurança serão respondidos em até 48 horas
- Trabalharemos com você para entender e resolver o problema
- Você será creditado na divulgação (a menos que prefira permanecer anônimo)
- Divulgação pública ocorrerá apenas após uma correção estar disponível

### Recomendações de Segurança

1. **Mantenha Python atualizado** - Use Python 3.8 ou superior
2. **Atualize dependências regularmente** - Execute `pip install --upgrade -r requirements.txt`
3. **Use a versão mais recente** - Baixe EchoX da seção [Releases](https://github.com/akra2017linux-rgb/EchoX/releases)
4. **Execute apenas código de fontes confiáveis** - Compile apenas a partir deste repositório oficial

### Dependências Seguras

Todas as dependências do EchoX são bibliotecas populares e bem mantidas:
- ✅ **CustomTkinter** - Framework UI mantido ativamente
- ✅ **Pygame** - Engine de áudio confiável e amplamente usado
- ✅ **Mutagen** - Biblioteca de metadados de áudio padrão da indústria
- ✅ **PyInstaller** - Ferramenta padrão para empacotamento Python
